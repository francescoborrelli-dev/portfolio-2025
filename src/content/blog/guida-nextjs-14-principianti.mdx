# Guida Completa a Next.js 14 per Principianti

Next.js si è affermato come uno dei framework React più popolari per lo sviluppo di applicazioni web moderne. Con il rilascio della versione 14, il framework introduce nuove funzionalità che rendono lo sviluppo ancora più efficiente e piacevole.

## Perché Scegliere Next.js?

Next.js risolve molti dei problemi comuni dello sviluppo React:

### 1. **Rendering Ibrido**
- Server-side rendering (SSR) out-of-the-box
- Static site generation (SSG) per performance ottimali
- Client-side rendering quando necessario

### 2. **Routing Semplificato**
- File-system based routing
- Nested routes automatiche
- Dynamic routes intuitive

### 3. **Ottimizzazioni Automatiche**
- Code splitting automatico
- Image optimization integrata
- Font optimization
- Bundle optimization

## Iniziare con Next.js 14

### Installazione

```bash
npx create-next-app@latest mio-progetto
cd mio-progetto
npm run dev
```

### Struttura del Progetto

```
mio-progetto/
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── globals.css
├── public/
├── package.json
└── next.config.js
```

## Novità di Next.js 14

### App Router (Stable)
Il nuovo App Router è ora stabile e offre:
- Layouts condivisi
- Nested routing
- Server Components di default
- Streaming e Suspense

### Server Actions
Funzioni che girano sul server e possono essere chiamate direttamente dal client:

```tsx
async function createUser(formData: FormData) {
  'use server'
  
  const name = formData.get('name')
  // Logica di creazione utente...
}

export default function UserForm() {
  return (
    <form action={createUser}>
      <input name="name" type="text" required />
      <button type="submit">Crea Utente</button>
    </form>
  )
}
```

### Turbopack (Beta)
Il nuovo bundler scritto in Rust:
- 10x più veloce di Webpack
- Hot reload istantaneo
- Bundle size ridotto

## Best Practices

### 1. Organizzazione dei File
```
app/
├── (marketing)/
│   ├── about/
│   └── contact/
├── dashboard/
│   ├── settings/
│   └── analytics/
└── api/
    └── users/
```

### 2. Gestione dello Stato
Per progetti semplici, usa useState e useContext. Per applicazioni complesse, considera Zustand o Redux Toolkit.

### 3. Performance
- Usa `Image` component per ottimizzazione automatica
- Implementa lazy loading con `Suspense`
- Utilizza Server Components quando possibile

## Deployment

### Vercel (Raccomandato)
```bash
npm install -g vercel
vercel
```

### Docker
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package.json ./
RUN npm install
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["npm", "start"]
```
